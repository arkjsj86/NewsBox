# e스포츠 탭 설계안

## 목적

이 문서는 NewsBox에 `e스포츠` 탭을 추가할 때의 기준을 정리한다.

이번 설계의 최우선 조건은 아래 두 가지다.

- 완전 무료로 운영할 것
- 외부 소스 상태가 흔들려도 사이트가 깨지지 않을 것

## 기본 방향

- 탭 이름은 `e스포츠`
- 탭 위치는 `Unity` 바로 앞
- 구현 후 기본 탭 순서는 `AI / 게임 / 연예 / e스포츠 / Unity`
- 이후 새 탭이 추가되더라도 `Unity`는 마지막 탭으로 유지한다

## 데이터 원칙

### 1. 유료 API 사용 금지

- X API 같은 유료 또는 사용량 기반 과금 API는 사용하지 않는다
- Riot API 개발 키 기반 구조도 기본 운영 방식으로 사용하지 않는다

### 2. 무료 소스 우선

- e스포츠 기사: 기존 게임 계열 RSS 피드에서 e스포츠 기사만 재분류
- 스포트라이트 일정: 공식 `LoL Esports` 일정 페이지 기반

### 3. 프론트는 정적 JSON만 읽는다

- 브라우저는 외부 사이트를 직접 호출하지 않는다
- GitHub Actions가 주기적으로 데이터를 수집해 JSON을 생성한다
- 화면은 생성된 JSON만 읽으므로 외부 사이트가 잠시 흔들려도 UI가 직접 깨지지 않는다

## 예상 데이터 구조

### 기사 목록

- `data/tabs/esports.json`
- 최근 e스포츠 기사 최대 20~30개 저장
- 기사 항목은 기존 탭과 동일하게 `title`, `url`, `source`, `publishedAt`, `summary` 중심으로 유지

### 스포트라이트 일정

- `data/spotlights/esports.json`
- 스포트라이트에는 다음 경기 1건만 저장
- 예시 필드:
  - `league`
  - `headline`
  - `matchLabel`
  - `scheduledAt`
  - `stage`
  - `format`
  - `sourceUrl`
  - `status`

## e스포츠 기사 분류 기준

기존 게임 RSS에서 아래 신호가 강한 기사만 `e스포츠` 탭으로 분류한다.

- LCK, LPL, LEC, LCS
- MSI, Worlds, 월즈
- 챔피언스, 리그, 플레이오프, 결승, 선발전
- T1, Gen.G, DK, KT, 한화생명e스포츠 등 팀명
- 밴픽, 경기 결과, 로스터, 이적, 감독, 코치, 선수 인터뷰
- 발로란트, 오버워치, 배틀그라운드 e스포츠 기사 중 경쟁전/리그 중심 기사

아래는 제외 우선 대상으로 본다.

- 일반 게임 업데이트 기사
- 단순 게임 출시/사전예약 기사
- 하드웨어 홍보 기사
- 게임 산업 일반 기사

## 스포트라이트 목적

`e스포츠` 탭의 스포트라이트는 일반 기사 1건이 아니라 `다음 주요 경기 일정`을 고정 노출하는 용도다.

표시 형태 예시:

- 제목: `다음 LCK 경기`
- 본문: `T1 vs Gen.G`
- 보조 정보: `2026-04-12 19:00 KST · LCK Week 2 · Bo3`

## 스포트라이트 전환 규칙

스포트라이트는 아래 우선순위로 다음 일정을 선택한다.

1. `LCK`
2. `MSI`
3. `Worlds`

세 리그/대회 중 현재 시점 이후의 `다음 경기`가 확인되면 우선순위에 따라 하나를 선택한다.

### 기본 규칙

- LCK 다음 경기가 있으면 무조건 LCK를 우선 노출한다
- LCK 다음 경기가 없고 MSI 일정이 있으면 MSI를 노출한다
- LCK와 MSI 다음 경기가 모두 없고 Worlds 일정이 있으면 Worlds를 노출한다

### 의도

- 국내 이용자 기준 기본 관심사는 LCK로 둔다
- 국제전 기간처럼 LCK가 비는 구간에는 MSI 또는 Worlds가 자동으로 대체한다
- 불필요하게 여러 대회를 동시에 섞어 보여주지 않는다

## 일정 수집 방식

- 공식 `LoL Esports` 일정 페이지에서 LCK, MSI, Worlds의 다음 경기 정보를 읽는다
- 각 대회별로 다음 경기 1건만 추출한다
- 최종적으로 우선순위 규칙을 적용해 스포트라이트 JSON 1건을 만든다

## 안정성 규칙

이 설계에서 가장 중요한 것은 사이트가 깨지지 않는 것이다.

### 실패 시 원칙

- 일정 추출 실패 시 페이지 전체를 비우지 않는다
- 마지막으로 성공한 `data/spotlights/esports.json`이 있으면 그대로 유지한다
- 마지막 성공 데이터도 없으면 안전한 기본 문구를 사용한다

### 기본 문구 예시

- 제목: `다음 주요 e스포츠 경기 준비 중`
- 본문: `현재 확인 가능한 LCK, MSI, Worlds 일정을 준비하고 있습니다.`

## 갱신 주기

- 기존 NewsBox와 동일하게 `3시간마다 정각`
- 기사 데이터와 스포트라이트 일정을 같은 워크플로에서 갱신한다

## 구현 우선순위

1. `e스포츠` 탭 추가
2. 게임 RSS에서 e스포츠 기사 재분류
3. `data/tabs/esports.json` 생성
4. `data/spotlights/esports.json` 생성
5. 스포트라이트를 기사형이 아닌 일정형으로 렌더링
6. 실패 시 fallback 유지

## 제외 범위

초기 MVP에서는 아래 항목을 하지 않는다.

- 실시간 중계 데이터
- 승부 예측
- 팀 순위표
- 선수 상세 전적
- 유료 API 연동
- X 연동
- 개인화 추천

## 한 줄 정리

`e스포츠` 탭은 무료 RSS 기반 기사와 공식 LoL Esports 일정 페이지를 조합해, `LCK 우선 / MSI 보조 / Worlds 보조` 규칙으로 스포트라이트를 구성하는 안정성 우선 설계로 진행한다.
