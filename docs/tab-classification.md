# NewsBox 탭 분류 기준

## 목적

이 문서는 NewsBox의 5개 탭에 어떤 뉴스가 들어가야 하는지에 대한 기준을 정리한다.

초기 버전에서는 한 기사가 여러 탭에 동시에 중복 노출되지 않도록, 기사마다 `대표 탭 1개`를 우선 지정하는 방식을 기본으로 한다.

추후 필요하면 내부적으로는 보조 태그를 둘 수 있지만, 사용자 화면에는 대표 탭 기준으로 노출하는 것을 원칙으로 한다.

## 탭 목록

- AI
- Unity
- 게임 산업
- 게임 개발
- 게임 일반

## 탭 정의

### AI

AI 자체가 중심 주제인 뉴스를 모은다.

포함 예시:

- 생성형 AI 모델 발표
- AI 기업 투자, 인수, 정책, 규제
- AI 기술 업데이트
- AI 서비스 출시
- AI 반도체, AI 플랫폼, AI 연구 동향

제외 기준:

- 게임 업계의 실적이나 인수합병이 핵심이면 `게임 산업`
- 게임 제작 과정의 기술 이야기라면 `게임 개발`
- Unity가 핵심 제품이나 플랫폼으로 등장하면 `Unity`

### Unity

Unity 엔진, Unity 생태계, Unity 회사 관련 뉴스를 모은다.

포함 예시:

- Unity 엔진 버전 업데이트
- Unity 정책 변경
- Unity Asset Store, 패키지, 툴링 소식
- Unity 관련 공식 발표
- Unity 기반 개발 워크플로우 변화

제외 기준:

- 단순히 게임이 Unity로 만들어졌다는 사실만으로는 `Unity`로 분류하지 않는다.
- Unity 언급이 보조적이고 핵심이 시장, 매출, 투자라면 `게임 산업`
- Unity가 아닌 일반적인 게임 개발 기술론이면 `게임 개발`

### 게임 산업

게임 비즈니스와 시장 흐름이 중심인 뉴스를 모은다.

포함 예시:

- 게임사 실적 발표
- 인수합병, 투자, 구조조정, 조직 개편
- 플랫폼 정책, 유통, 퍼블리싱 계약
- 국가별 규제, 법률, 확률형 아이템 정책
- 콘솔, 스토어, 마켓 점유율 변화

제외 기준:

- 개발 기술이나 제작 방식이 핵심이면 `게임 개발`
- 플레이어 대상 일반 소식이면 `게임 일반`
- Unity 자체가 핵심이면 `Unity`

### 게임 개발

게임을 만드는 과정과 기술이 중심인 뉴스를 모은다.

포함 예시:

- 게임 프로그래밍, 렌더링, 그래픽스, 최적화
- 툴체인, 파이프라인, QA, 빌드 자동화
- 엔진 비교, 개발 워크플로우, 기술 포스트모템
- 아트 파이프라인, 레벨 디자인, 서버 구조
- 게임 개발자 대상 기술 발표

제외 기준:

- Unity가 핵심 제품이나 플랫폼이면 `Unity`
- 비즈니스, 시장, 정책이 핵심이면 `게임 산업`
- 일반 이용자 관점의 출시, 이벤트, 리뷰 소식이면 `게임 일반`

### 게임 일반

게임 이용자 관점의 일반 뉴스와 대중적 이슈를 모은다.

포함 예시:

- 신작 발표
- 출시 일정
- 대규모 업데이트
- 트레일러 공개
- 이벤트, e스포츠, 화제성 이슈
- 특정 게임 커뮤니티 반응이나 일반 이용자 관심사

제외 기준:

- 회사 실적, 규제, 계약, 투자 중심이면 `게임 산업`
- 개발 기술 중심이면 `게임 개발`
- Unity 중심 기사면 `Unity`

## 분류 우선 규칙

기사가 여러 탭에 걸쳐 보일 수 있을 때는 아래 순서로 대표 탭을 결정한다.

1. Unity가 핵심 주제이면 `Unity`
2. 게임 회사, 시장, 정책, 투자 이야기면 `게임 산업`
3. 게임 제작 기술과 워크플로우 이야기면 `게임 개발`
4. AI 자체가 핵심 주제이면 `AI`
5. 위 조건에 뚜렷하게 속하지 않는 게임 관련 일반 뉴스는 `게임 일반`

## 혼동되기 쉬운 사례

### AI와 게임 개발이 함께 나오는 경우

- 기사의 초점이 "AI 기술 자체의 변화"라면 `AI`
- 기사의 초점이 "게임 개발 과정에서 AI를 어떻게 쓰는가"라면 `게임 개발`

### Unity와 게임 개발이 함께 나오는 경우

- Unity 기능, Unity 업데이트, Unity 정책이 핵심이면 `Unity`
- Unity는 예시일 뿐이고 일반적인 개발 방식 설명이 핵심이면 `게임 개발`

### 게임 산업과 게임 일반이 함께 나오는 경우

- 기업 발표, 실적, 규제, 계약, 시장 영향이 핵심이면 `게임 산업`
- 이용자 입장에서 소비하는 일반 소식이면 `게임 일반`

## 초기 운영 원칙

- 한 기사는 대표 탭 1개만 가진다.
- 제목과 본문 요약을 함께 보고 분류한다.
- 탭 분류가 애매하면 가장 넓은 탭으로 무조건 보내지 않고, 기사 중심 주제를 다시 확인해 대표 탭을 정한다.
- 같은 기사가 여러 탭에 중복 노출되는 일은 최대한 줄인다.

## 추후 확장 가능성

향후 아래 항목을 추가할 수 있다.

- 보조 태그 시스템
- 기사 신뢰도 점수
- 출처별 가중치
- 탭별 수집 규칙 파일 분리
- 관리자용 분류 보정 규칙
