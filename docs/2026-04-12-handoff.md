# 2026/04/12 작업 메모

## 현재 상태

- 작업 브랜치: `feature/jhpark_home`
- 연예 탭 우측 패널 작업은 4단계로 쪼개서 진행 중
- 1단계 `데이터만 이슈형으로 바꾸기` 완료
- 아직 커밋하지 않음

## 오늘 완료한 내용

- 연예 스포트라이트 데이터를 기사형에서 이슈형으로 전환함
- 파일: `scripts/lib/entertainment-trends.mjs`
- 생성 결과 파일: `data/spotlights/entertainment.json`

현재 `items` 구조는 아래 필드를 가짐:

- `issueKey`
- `issueTitle`
- `articleCount`
- `relatedArticles`
- `representativeArticle`

예시:

- `한남더힐아파트` 이슈 아래에 관련 기사 여러 건이 묶임
- 대표 기사는 `representativeArticle`로 따로 저장됨

## 내일 바로 시작할 작업

### 2단계

우측 패널이 이슈형 JSON을 읽도록 변경

작업 포인트:

- `src/app.js`
- 현재 연예 패널 렌더러가 기사형 필드를 직접 읽고 있음
- 이제 `item.title` 대신 `item.issueTitle` 중심으로 표시
- 클릭 링크는 우선 `representativeArticle.url` 사용
- `relatedArticles.length` 또는 `articleCount`를 같이 보여주기

목표:

- 같은 이슈가 여러 줄로 반복되지 않고
- `이슈 1개 = 패널 1줄`로 보이게 만들기

## 그 다음 작업

### 3단계

대표 기사/관련 기사 표시 다듬기

- 대표 기사 제목을 보조 텍스트로 보여줄지 결정
- `관련 기사 n건` 문구 추가
- 출처 표기는 대표 기사 기준으로 정리

### 4단계

히스토리/품질 보정

- 현재 연예가 아닌 이슈가 섞일 수 있음
- 예: `박성한` 같은 비연예 이슈가 들어왔던 사례 있음
- 연예 필터 정밀도 보정 필요
- 같은 이슈 제목 정규화 방식도 추가 점검 필요

## 주의할 점

- 현재 워크트리에 생성 파일 변경이 함께 섞여 있음
- 특히 아래 미추적 파일 존재:
  - `data/metadata_BACKUP_1614.json`
  - `data/metadata_BASE_1614.json`
  - `data/metadata_LOCAL_1614.json`
  - `data/metadata_REMOTE_1614.json`
- 이 파일들은 내가 만든 작업 산출물이 아니므로 삭제 전 확인 필요
